# Administrative dashboard family

Reference: `design-references/weelmat-admin-dashboard-concept.png`.

- Lead with page title, context, primary action, summary strip, and items requiring attention.
- Desktop uses a submissions table with a narrow exception/completion rail.
- Mobile uses stacked records and filter sheet; never force an unreadable desktop table.
- Review uses a focused drawer/dialog with status, notes, approve, and request-revision actions.
- Public status reuses the reading hierarchy but removes private fields and edit actions.
